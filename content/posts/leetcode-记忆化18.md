## 吃掉 N 个橘子的最少天数

厨房里总共有 n 个橘子，你决定每一天选择如下方式之一吃这些橘子：

吃掉一个橘子。
如果剩余橘子数 n 能被 2 整除，那么你可以吃掉 n/2 个橘子。
如果剩余橘子数 n 能被 3 整除，那么你可以吃掉 2*(n/3) 个橘子。

每天你只能从以上 3 种方案中选择一种方案。

请你返回吃掉所有 n 个橘子的最少天数。

```
use std::collections::HashMap;

impl Solution {
    /// 计算吃掉 `n` 个橘子所需的最少天数。
    /// 每天可以执行以下三种操作之一：
    /// 1. 吃掉 1 个橘子。
    /// 2. 如果橘子数能被 2 整除，可以吃掉一半（即剩余 n/2）。
    /// 3. 如果橘子数能被 3 整除，可以吃掉三分之二（即剩余 n/3）。
    ///
    /// 该问题等价于：给定初始数字 n，每次可以减 1，或当 n 为偶数时除以 2，
    /// 或当 n 为 3 的倍数时除以 3，求变成 0 的最少操作次数。
    /// 注意：除以 2 或除以 3 本身算作一次操作，但会跳过中间的大量减 1 操作。
    ///
    /// 算法采用记忆化深度优先搜索（DFS），自顶向下递归，利用 HashMap 缓存已计算的状态。
    /// 时间复杂度约 O(log^2 n) 级别，可处理 n 高达 2e9 的输入。
    pub fn min_days(n: i32) -> i32 {
        let mut memo = HashMap::new();
        Self::dfs(n, &mut memo)
    }

    /// 递归函数，返回将 `i` 个橘子吃完所需的最少天数。
    /// `memo` 用于缓存中间结果，避免重复计算。
    fn dfs(i: i32, memo: &mut HashMap<i32, i32>) -> i32 {
        // 基础情况：0 或 1 个橘子时，天数就是橘子数本身（吃 0 天或 1 天）。
        if i <= 1 {
            return i;
        }

        // 如果已经计算过，直接返回缓存值。
        if let Some(&cached) = memo.get(&i) {
            return cached;
        }

        // 核心递推公式：
        // - 我们可以先通过若干次“减 1”操作，使得当前橘子数能被 2 或 3 整除，
        //   然后进行一次整除操作，从而跳过大量重复的减 1。
        // - 对于除以 2 的情况：需要先执行 i % 2 次减 1，使 i 变为偶数，然后一次除以 2 操作，
        //   剩余橘子数为 i/2，总天数 = (i % 2) + 1 + dfs(i/2)。
        // - 对于除以 3 的情况同理：天数 = (i % 3) + 1 + dfs(i/3)。
        // - 最后取两种策略的较小值。
        let res = (Self::dfs(i / 2, memo) + i % 2)
            .min(Self::dfs(i / 3, memo) + i % 3)
            + 1;

        // 记忆化存储
        memo.insert(i, res);
        res
    }
}
```
