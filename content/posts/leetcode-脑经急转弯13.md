## 得到整数零需要执行的最少操作数

给你两个整数：num1 和 num2 。

在一步操作中，你需要从范围 [0, 60] 中选出一个整数 i ，并从 num1 减去 2i + num2 。

请你计算，要想使 num1 等于 0 需要执行的最少操作数，并以整数形式返回。

如果无法使 num1 等于 0 ，返回 -1 。

```
impl Solution {
    pub fn make_the_integer_zero(num1: i32, num2: i32) -> i32 {
        if num2 > num1 {
            return -1;
        }

        let (n1, n2) = (num1 as i64, num2 as i64);

        (1..)
            .map(|k| (k, n1 - k * n2))
            .take_while(|&(_, rem)| rem >= 0)
            .find(|&(k, rem)| rem >= k && rem.count_ones() <= k as u32)
            .map(|(k, _)| k as i32)
            .unwrap_or(-1)
    }
}
```

将参数转换为 i64 类型。原因：

后续计算 k * n2 可能溢出 32 位整数
i64 提供更大的范围，避免溢出

“(1..)”
        创建一个从 1 开始的无限迭代器：1, 2, 3, 4, ...


“.map(|k| (k, n1 - k * n2))”
        将每个 k 映射为一个元组 (k, remaining)，其中 remaining = n1 - k * n2。

        示例：如果 n1=10, n2=3

        k=1 → (1, 10-3=7)
        k=2 → (2, 10-6=4)
        k=3 → (3, 10-9=1)
        k=4 → (4, 10-12=-2)
        ...
        现在迭代器包含：(1,7), (2,4), (3,1), (4,-2), ...


“.take_while(|&(_, rem)| rem >= 0)”
       take_while 从迭代器中取值，直到条件不满足为止。

       |&(_, rem)| 是一个闭包：

       & 表示引用
       (_, rem) 是模式匹配，忽略元组的第一个元素，只关注 rem
       条件：rem >= 0
       效果：只保留 remaining >= 0 的元素，一旦遇到负数就停止。

       继续上面的例子：

       (1,7): 7>=0 ✓ 保留
       (2,4): 4>=0 ✓ 保留
       (3,1): 1>=0 ✓ 保留
       (4,-2): -2>=0 ✗ 停止，不包含这个元素
       现在迭代器只包含：(1,7), (2,4), (3,1)




“.find(|&(k, rem)| rem >= k && rem.count_ones() <= k as u32)”
       find 在迭代器中查找第一个满足条件的元素。

       条件分解：

       rem >= k：剩余值至少等于操作次数（每次操作至少能处理1）
       rem.count_ones()：计算 rem 的二进制表示中 1 的个数
       <= k as u32：1 的个数不超过操作次数
       示例继续：

       (1,7): 检查 7>=1 ✓, 7的二进制111有3个1, 3<=1? ✗ → 不满足
       (2,4): 检查 4>=2 ✓, 4的二进制100有1个1, 1<=2? ✓ → 满足！
       所以 find 返回 Some((2,4))




“.map(|(k, _)| k as i32)”
       如果 find 找到了值，map 提取元组的第一个元素（即 k），并转换为 i32。
       继续例子：Some((2,4)) → Some(2)




“.unwrap_or(-1)”
       如果前面的结果是 Some(value)，返回 value
       如果是 None（没找到符合条件的），返回 -1
       继续例子：Some(2) → 返回


关键函数说明

map：转换每个元素
take_while：提前终止迭代
find：查找第一个匹配项
unwrap_or：安全地解包 Option
