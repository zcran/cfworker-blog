## 我能赢吗

在 "100 game" 这个游戏中，两名玩家轮流选择从 1 到 10 的任意整数，累计整数和，先使得累计整数和 达到或超过  100 的玩家，即为胜者。

如果我们将游戏规则改为 “玩家 不能 重复使用整数” 呢？

例如，两个玩家可以轮流从公共整数池中抽取从 1 到 15 的整数（不放回），直到累计整数和 >= 100。

给定两个整数 maxChoosableInteger （整数池中可选择的最大数）和 desiredTotal（累计和），若先出手的玩家能稳赢则返回 true ，否则返回 false 。假设两位玩家游戏时都表现 最佳 。



```
impl Solution {
    pub fn can_i_win(max_choosable_integer: i32, desired_total: i32) -> bool {
        // ==================== 边界条件处理 ====================
        // 如果目标总和为0，先手玩家什么都不用做就赢了
        // 这是边界情况的防御性编程
        if desired_total == 0 { return true; }

        // ==================== 数学剪枝优化 ====================
        let n = max_choosable_integer;
        // n * (n + 1) / 2 是 1+2+...+n 的和（等差数列求和公式）
        // 如果所有数字加起来都达不到目标，先手玩家无论如何都会输
        // 例如：n=5, desired_total=16，最大和=15<16 → 直接返回 false
        if n * (n + 1) / 2 < desired_total { return false; }

        // ==================== 状态空间初始化 ====================
        // 将 n 转换为 usize 类型，用于位运算
        let n_usize = n as usize;
        // 1 << n_usize 计算状态总数 = 2^n
        // 例如 n=3 时，有 8 种状态（000,001,010,...,111）
        // vec![0i8; 状态数] 创建记忆化数组：
        //   0i8 = 0: 表示该状态未计算
        //   1i8 = 1: 表示该状态当前玩家能赢
        //   -1i8 = -1: 表示该状态当前玩家必输
        let mut memo = vec![0i8; 1 << n_usize];

        // ==================== 递归函数定义 ====================
        // 参数说明：
        //   mask: 位掩码，第 i 位为 1 表示数字 i+1 已被使用
        //   total: 还需要达到的总和（从 desired_total 递减）
        //   n: 最大可选数字
        //   memo: 记忆化数组的可变引用
        fn dfs(mask: usize, total: i32, n: usize, memo: &mut [i8]) -> bool {
            // ==================== 递归终止条件 ====================
            // 如果 total ≤ 0，说明上一轮对手选完后已经达到或超过目标
            // 当前玩家还没有机会选数就已经输了
            // 返回 false 表示当前玩家无法获胜
            if total <= 0 { return false; }

            // 如果该状态已经计算过（memo[mask] ≠ 0）
            // memo[mask] == 1 返回 true，否则返回 false
            // 这是记忆化搜索的核心，避免重复计算
            if memo[mask] != 0 { return memo[mask] == 1; }

            // ==================== 核心博弈逻辑 ====================
            // 遍历所有可能的数字（i=0 代表数字 1，i=1 代表数字 2，...）
            for i in 0..n {
                // 创建第 i 位的掩码
                // 例如 i=0 → bit=1 (二进制 001)
                // i=1 → bit=2 (二进制 010)
                // i=2 → bit=4 (二进制 100)
                let bit = 1 << i;

                // 检查数字 i+1 是否未被使用
                // 按位与运算，如果结果为 0 表示该位为 0，数字可用
                if mask & bit == 0 {
                    // 递归调用，让对手面对新的状态：
                    // mask | bit: 将当前数字标记为已使用
                    // total - (i as i32 + 1): 减去当前选择的数字（i+1）
                    // !dfs(...): 如果对手在下一状态返回 false（即对手会输）
                    //           那么当前玩家选择这个数字就能赢
                    if !dfs(mask | bit, total - (i as i32 + 1), n, memo) {
                        memo[mask] = 1;  // 标记当前状态为必胜
                        return true;
                    }
                }
            }

            // ==================== 无路可走的情况 ====================
            // 如果循环结束都没找到能赢的选择
            // 说明无论当前玩家选什么数字，对手都能赢
            memo[mask] = -1;  // 标记当前状态为必输
            false
        }

        // ==================== 初始调用 ====================
        // mask = 0: 初始状态，所有数字都未使用
        // total = desired_total: 需要达到的目标总和
        // 返回先手玩家是否能赢
        dfs(0, desired_total, n_usize, &mut memo)
    }
}
```

复杂度分析

时间复杂度: O(n × 2ⁿ)，其中 n = max_choosable_integer

状态数: 2ⁿ
每个状态尝试 O(n) 个数字
空间复杂度: O(2ⁿ)

memo 数组大小
递归调用栈深度 O(n)
关键技巧总结

位掩码: 用整数表示哪些数字被选过，空间效率 O(2ⁿ)
记忆化搜索: 避免重复计算相同状态
博弈论转换: !dfs(...) 表示对手会输
递减 total: 从目标值递减，比递增更容易判断胜负
数学剪枝: 提前排除不可能的情况
