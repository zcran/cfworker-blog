## 计异或值在范围内的数对有多少

给你一个整数数组 nums （下标 从 0 开始 计数）以及两个整数：low 和 high ，请返回 漂亮数对 的数目。

漂亮数对 是一个形如 (i, j) 的数对，其中 0 <= i < j < nums.length 且 low <= (nums[i] XOR nums[j]) <= high 。

这是一个非常巧妙的解法！让我逐行详细注释：

```
impl Solution {
    pub fn count_pairs(nums: Vec<i32>, low: i32, high: i32) -> i32 {
        // 题目：统计满足 low <= (nums[i] XOR nums[j]) <= high 的数对数量（i < j）

        // 统计每个数字出现的次数
        let mut cnt = std::collections::HashMap::<i32, i32>::new();
        for num in nums {
            *cnt.entry(num).or_default() += 1;
        }

        let mut ans = 0;
        let mut low = low;
        let mut high = high + 1;  // 将区间 [low, high] 转为 [low, high+1)，便于位运算

        // 核心思想：通过二进制位逐层处理，类似数字 DP（数位动态规划）
        // 每次处理当前最低位，然后将所有数字右移，继续处理高位
        while low > 0 || high > 0 {
            // 新的一层哈希表，存储右移后的数字计数
            let mut ncnt = std::collections::HashMap::<i32, i32>::new();

            // 遍历当前层的所有数字及其计数
            for (&n, &cn) in &cnt {
                // n >> 1：去掉最低位，进入下一层
                // 相同父节点的数字会合并在一起
                *ncnt.entry(n >> 1).or_default() += cn;

                // ========== 关键：统计异或值在第 k 位为 1 的数对 ==========
                // 当前正在处理第 k 位（k 从 0 开始）
                // high 的最低位表示：在已处理的高位相同的情况下，当前位是否需要考虑

                if high & 1 == 1 {
                    // 当 high 当前位为 1 时：
                    // 说明异或值在这一位可以为 1，且依然 ≤ high（的高位部分）
                    // 需要统计所有与 n 异或后当前位为 1 的数字
                    // 即：n XOR other = 1 (当前位)
                    // => other = n XOR 1 (当前位)
                    // 注意：high - 1 的目的是将当前位设为 0，其他高位保持不变
                    ans += cn * cnt.get(&(n ^ (high - 1))).unwrap_or(&0);
                }

                if low & 1 == 1 {
                    // 当 low 当前位为 1 时：
                    // 说明异或值在这一位可以为 1，但需要减去 < low 的部分
                    // 因为我们要的是 ≥ low 的，所以要减去异或值 < low 的数对
                    // low - 1 是为了得到小于 low 的最大值
                    ans -= cn * (cnt.get(&(n ^ (low - 1)))).unwrap_or(&0);
                }
            }

            // 右移一位，处理下一位（更高位）
            high >>= 1;
            low >>= 1;
            cnt = ncnt;  // 进入下一层
        }

        // 每个数对被计算了两次（当 i 和 j 分别作为 n 时），所以除以 2
        ans / 2
    }
}
```

## 算法核心原理解析

### 1. 问题转换
统计满足 `low <= (a XOR b) <= high` 的数对数量。

等价于：统计 `(a XOR b) <= high` 的数量 - 统计 `(a XOR b) <= low-1` 的数量。

### 2. 核心技巧：位运算 + 逐层统计

假设我们要统计 `(a XOR b) <= limit` 的数量：

- 从最高位到最低位逐位处理
- 使用类似 Trie 的方式，但这里用哈希表实现
- 每次处理当前位后，将所有数字右移，相当于进入下一层（去除已处理的低位）

### 3. 关键理解：`n ^ (high - 1)` 的含义

假设 `high` 的二进制是 `10110`，当前正在处理最低位（第0位）：
- `high - 1 = 10101`，将最低位变为0，其他位不变
- `n ^ (high - 1)`：与 `n` 异或后，在当前位的结果是 `n_bit ^ 1`（因为 `(high-1)` 的当前位为0？需要仔细理解）

实际上这个技巧的精髓在于：
- 当 `high` 的当前位为 1 时，意味着异或值的这一位可以取 0 或 1
- 但为了保持整体 ≤ high，需要分情况讨论
- `n ^ (high - 1)` 计算的是：使得异或值与 high 的高位完全相同的另一数字

### 4. 示例演示

假设 `nums = [1, 2, 3]`，统计 `1 <= XOR <= 2` 的数对：

```
1 XOR 2 = 3（不满足）
1 XOR 3 = 2（满足）
2 XOR 3 = 1（满足）
```

算法执行过程（简化版）：

```
初始：cnt = {1:1, 2:1, 3:1}
low=1 (001), high=3 (011)  // high+1=3

第0层：
处理最低位...
ans = 1（统计结果）
然后右移：cnt变成 {0:2, 1:1}  // 1>>1=0, 2>>1=1, 3>>1=1
high=1, low=0

第1层：
处理下一位...
ans 更新...
最终 ans/2 = 2
```

## 算法复杂度

- **时间复杂度**：O(N × log M)，其中 M 是数字的最大值（32），实际为 O(32N)
- **空间复杂度**：O(N)，每层哈希表的大小递减

## 巧妙之处

1. **不需要显式构建 Trie**：通过哈希表和逐层右移实现类似效果
2. **一次性统计所有数对**：不需要双重循环
3. **位运算技巧精湛**：`high-1` 和 `low-1` 的妙用
4. **边统计边压缩**：每处理完一层，数字右移，数量合并

这是一个非常优雅且高效的解法，充分体现了位运算和分治思想！
