##  不同路径


一个机器人位于一个 m x n 网格的左上角 （起始点在下图中标记为 “Start” ）。

机器人每次只能向下或者向右移动一步。机器人试图达到网格的右下角（在下图中标记为 “Finish” ）。

问总共有多少条不同的路径？


```
impl Solution {
    /// 计算从网格左上角走到右下角的不同路径数（只能向右或向下移动）
    /// 时间复杂度 O(m*n)，空间复杂度 O(m*n)（可优化为 O(n)，见下方说明）
    pub fn unique_paths(m: i32, n: i32) -> i32 {
        let (m, n) = (m as usize, n as usize);

        // 1. 创建二维动态规划数组
        // dp[i][j] 表示从起点 (0,0) 走到 (i,j) 的不同路径数
        let mut dp = vec![vec![0; n]; m];

        // 2. 初始化边界条件
        // 第一行：只能从左边过来，所以只有一种路径（一直向右）
        for i in 0..n {
            dp[0][i] = 1;
        }
        // 第一列：只能从上边过来，所以只有一种路径（一直向下）
        for i in 0..m {
            dp[i][0] = 1;
        }

        // 3. 动态规划状态转移
        // 走到 (i,j) 可以从上方 (i-1,j) 向下移动一格，或从左方 (i,j-1) 向右移动一格
        for i in 1..m {
            for j in 1..n {
                dp[i][j] = dp[i - 1][j] + dp[i][j - 1];
            }
        }

        // 4. 返回终点的路径数（右下角）
        dp[m - 1][n - 1]
    }
}
```
