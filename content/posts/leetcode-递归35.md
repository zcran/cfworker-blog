## 交替删除操作后最后剩下的整数


给你一个整数 n。

我们将 1 到 n 的整数按从左到右的顺序排成一个序列。然后，交替 地执行以下两种操作，直到只剩下一个整数为止，从操作 1 开始：

操作 1：从左侧开始，隔一个数删除一个数。
操作 2：从右侧开始，隔一个数删除一个数。

返回最后剩下的那个整数。


```
impl Solution {
    /// 计算经过重复删除操作后最后剩下的整数
    ///
    /// # 问题说明
    /// 有一个从 1 到 n 的整数序列，重复执行以下操作直到只剩一个数：
    /// 1. 从左到右删除所有奇数位置的数
    /// 2. 从右到左删除所有奇数位置的数
    /// 3. 重复步骤 1 和 2
    ///
    /// # 数学推导
    /// 这个问题有数学公式解：
    /// - 每次操作后，剩余数字形成一个等差数列
    /// - 通过跟踪首项和公差，可以在 O(log n) 时间内计算出结果
    ///
    /// # 复杂度
    /// - 时间复杂度：O(log n) - 每次循环 n 减半
    /// - 空间复杂度：O(1) - 只使用常数个变量
    ///
    /// # 参数
    /// - `n`: 初始序列的最大数字
    ///
    /// # 返回
    /// - 最后剩下的整数
    ///
    /// # 示例
    /// ```
    /// assert_eq!(Solution::last_integer(9), 6);
    /// // 过程：1,2,3,4,5,6,7,8,9
    /// // 第1次(左->右): 2,4,6,8 (删除奇数位:1,3,5,7,9)
    /// // 第2次(右->左): 2,6 (删除奇数位从右:8,4)
    /// // 第3次(左->右): 6 (删除奇数位:2)
    /// ```
    pub fn last_integer(n: i64) -> i64 {
        let mut start = 1;      // 等差数列的首项
        let mut diff = 1;       // 等差数列的公差（可以是负数）
        let mut remaining = n;   // 当前剩余数字的个数

        while remaining > 1 {
            // 更新首项：根据剩余数字个数的奇偶性决定是否跳过第一个数
            // n-2 + n%2 的含义：
            // - 当 n 为偶数时，n%2=0，跳过的数量为 n-2
            // - 当 n 为奇数时，n%2=1，跳过的数量为 n-1
            start += (remaining - 2 + remaining % 2) * diff;

            // 公差每次取反并加倍（方向交替，步长加倍）
            diff *= -2;

            // 剩余数字个数每次减半（向上取整）
            remaining = (remaining + 1) / 2;
        }

        start
    }
}
```
