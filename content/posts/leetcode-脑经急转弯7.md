## 所有蚂蚁掉下来前的最后一刻

有一块木板，长度为 n 个 单位 。一些蚂蚁在木板上移动，每只蚂蚁都以 每秒一个单位 的速度移动。其中，一部分蚂蚁向 左 移动，其他蚂蚁向 右 移动。

当两只向 不同 方向移动的蚂蚁在某个点相遇时，它们会同时改变移动方向并继续移动。假设更改方向不会花费任何额外时间。

而当蚂蚁在某一时刻 t 到达木板的一端时，它立即从木板上掉下来。

给你一个整数 n 和两个整数数组 left 以及 right 。两个数组分别标识向左或者向右移动的蚂蚁在 t = 0 时的位置。请你返回最后一只蚂蚁从木板上掉下来的时刻。


```
impl Solution {
    pub fn get_last_moment(n: i32, left: Vec<i32>, right: Vec<i32>) -> i32 {
        left.into_iter()
            .chain(right.into_iter().map(|x| n - x))
            .max()
            .unwrap()
    }
}
```


这是 LeetCode 第 1503 题（所有蚂蚁掉下来前的最后一刻）的题解代码。这段代码非常简洁，但背后有深刻的物理/数学原理。让我详细解析。

## 1. 问题理解

- 有一根长度为 `n` 的木棍，坐标从 0 到 n
- 若干蚂蚁在木棍上，每只蚂蚁以速度 1 向左或向右移动
- 当两只蚂蚁相遇时，它们会**立即掉头**
- 当蚂蚁到达木棍端点（0 或 n）时，就会掉落
- 问：**最后一只蚂蚁掉落的时刻**是多少？

参数：
- `n`：木棍长度
- `left`：初始位置在左边且**向左**移动的蚂蚁（实际上题目描述是"向左移动的蚂蚁"，即它们朝向 0 点移动）
- `right`：初始位置在右边且**向右**移动的蚂蚁（朝向 n 点移动）

---

## 2. 核心洞察：相遇掉头等价于穿过

这个问题的关键技巧是：**两只蚂蚁相遇后掉头，等价于它们互相穿过继续前进**。

### 2.1 为什么可以这样等价？

因为蚂蚁是**不可区分的**：
- 如果两只蚂蚁 A 和 B 相遇后掉头，从外部观察，看到的是 A 沿着 B 原来的方向继续，B 沿着 A 原来的方向继续
- 这与它们"穿过"彼此继续前进的效果完全一样
- 我们只关心蚂蚁何时掉落，不关心具体哪只蚂蚁掉落

因此，我们可以把问题简化为：
- **每只蚂蚁都沿原方向直线运动，不会掉头**
- 这样，每只蚂蚁到达端点的时间就是它原本的方向决定的

## 3. 代码逐行解析

```rust
impl Solution {
    pub fn get_last_moment(n: i32, left: Vec<i32>, right: Vec<i32>) -> i32 {
        left.into_iter()
            .chain(right.into_iter().map(|x| n - x))
            .max()
            .unwrap()
    }
}
```

### 3.1 处理左边蚂蚁
```rust
left.into_iter()
```
- 左边蚂蚁（向左移动）需要走到 0 点
- 距离 = 当前位置（因为位置就是到左端点的距离）
- 时间 = 距离 / 速度 = 距离（速度=1）
- 所以直接使用 `left` 中的位置值

### 3.2 处理右边蚂蚁
```rust
right.into_iter().map(|x| n - x)
```
- 右边蚂蚁（向右移动）需要走到 n 点
- 距离 = n - 当前位置
- 时间 = n - x
- 通过 `map` 将每个右边蚂蚁的位置转换为掉落时间

### 3.3 合并并取最大值
```rust
.chain(...)  // 将左右两边的掉落时间合并成一个迭代器
.max()       // 找出最大的掉落时间（最后掉落的时刻）
.unwrap()    // 因为 left 和 right 可能都为空，但题目保证至少有一只蚂蚁？
```

---

## 4. 示例验证

### 示例 1：
```
n = 4, left = [4, 3], right = [0, 1]
```
- left 蚂蚁掉落时间：4, 3
- right 蚂蚁掉落时间：4-0=4, 4-1=3
- 所有时间：4, 3, 4, 3
- 最大值 = 4

### 示例 2：
```
n = 7, left = [], right = [0,1,2,3,4,5,6,7]
```
- left 无蚂蚁
- right 时间：7,6,5,4,3,2,1,0
- 最大值 = 7

### 示例 3：
```
n = 7, left = [0,1,2,3,4,5,6,7], right = []
```
- left 时间：0,1,2,3,4,5,6,7
- 最大值 = 7

---

## 5. 算法复杂度

- 时间复杂度：O(m + k)，其中 m = left.len()，k = right.len()
- 空间复杂度：O(1)，只使用迭代器，不额外分配空间

---

## 6. 为什么这样正确？

数学证明：
1. 每只蚂蚁掉落的时刻 = 它到目标端点的距离
2. 相遇掉头等价于穿过，所以蚂蚁的运动轨迹可以视为直线
3. 最后掉落的时刻 = max(所有蚂蚁的掉落时间)
4. 左边蚂蚁掉落时间 = 位置
5. 右边蚂蚁掉落时间 = n - 位置

因此，答案就是：
```
max( max(left), max(n - right) )
```
其中空集合的 max 视为负无穷（不影响结果）

## 7. 总结

这段代码的巧妙之处在于：
- 将复杂的碰撞掉头问题简化为直线运动
- 用一行代码解决了看似需要模拟的问题
- 展现了"问题转换"的思维魅力

这是一个典型的"脑筋急转弯"类算法题，理解了"相遇掉头 = 穿过"这个等价关系后，解决方案就变得极其简单。
