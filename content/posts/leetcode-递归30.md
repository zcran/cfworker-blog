## 猴子碰撞的方法数

现在有一个正凸多边形，其上共有 n 个顶点。顶点按顺时针方向从 0 到 n - 1 依次编号。每个顶点上 正好有一只猴子 。下图中是一个 6 个顶点的凸多边形。



每个猴子同时移动到相邻的顶点。顶点 i 的相邻顶点可以是：

顺时针方向的顶点 (i + 1) % n ，或
逆时针方向的顶点 (i - 1 + n) % n 。
如果移动后至少有两只猴子停留在同一个顶点上或者相交在一条边上，则会发生 碰撞 。

返回猴子至少发生 一次碰撞 的移动方法数。由于答案可能非常大，请返回对 10^9+7 取余后的结果。

注意，每只猴子只能移动一次。

```
impl Solution {
    /// 计算猴子移动的总方案数
    ///
    /// # 问题背景
    /// 这是一个经典问题：n 只猴子围成一圈，每只猴子可以向左或向右移动一个位置，
    /// 求所有猴子都不冲突（不占据同一位置）的移动方案总数。
    ///
    /// # 数学推导
    /// 经过分析，这个问题等价于：
    /// 1. 总方案数 = 2^n - 2（当 n > 1 时）
    /// 2. 所有猴子同时向左或同时向右会导致位置交换，但题目通常排除这种平凡解
    /// 3. 所以最终结果 = (2^n - 2) mod MOD
    ///
    /// # 为什么是 2^n - 2？
    /// - 每只猴子有 2 种选择（左或右），总共有 2^n 种方案
    /// - 减去所有猴子都向左的方案（1 种）
    /// - 减去所有猴子都向右的方案（1 种）
    /// - 剩余方案数为 2^n - 2
    ///
    /// # 复杂度
    /// - 时间复杂度：O(log n) - 快速幂
    /// - 空间复杂度：O(1)
    ///
    /// # 示例
    /// ```
    /// assert_eq!(Solution::monkey_move(1), 0);  // 2^1 - 2 = 0
    /// assert_eq!(Solution::monkey_move(2), 2);  // 2^2 - 2 = 2
    /// assert_eq!(Solution::monkey_move(3), 6);  // 2^3 - 2 = 6
    /// assert_eq!(Solution::monkey_move(4), 14); // 2^4 - 2 = 14
    /// ```
    pub fn monkey_move(n: i32) -> i32 {
        const MOD: i64 = 1000000007;

        // 使用快速幂计算 2^n mod MOD
        // fast 作为指数，从 n 开始
        let mut fast = n;

        // result 存储最终结果，初始化为 1（乘法的单位元）
        let mut result: i64 = 1;

        // base 是底数，初始为 2，表示计算 2 的幂
        let mut base: i64 = 2;

        // 快速幂算法（二进制指数分解）
        // 将指数 n 表示为二进制，例如 n=13 = 1101₂ = 8+4+1
        // 则 2^13 = 2^8 × 2^4 × 2^1
        while fast > 0 {
            // 检查当前二进制位是否为 1
            // fast % 2 == 1 等价于检查最低位是否为 1
            if fast % 2 == 1 {
                // 如果当前位是 1，将当前的 base 乘到结果中
                result = result * base % MOD;
            }

            // base 自身平方，准备下一位
            // 2^1 → 2^2 → 2^4 → 2^8 → ...
            base = base * base % MOD;

            // 右移一位，相当于除以 2
            // 处理下一个二进制位
            fast /= 2;
        }

        // 计算最终结果：(2^n - 2) mod MOD
        // 使用 (result - 2 + MOD) % MOD 避免负数
        // 加法 MOD 确保在减法结果为负数时能正确取模
        ((result - 2 + MOD) % MOD) as i32
    }
}
