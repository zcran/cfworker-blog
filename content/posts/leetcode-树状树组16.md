## 得到回文串的最少操作次数

给你一个只包含小写英文字母的字符串 s 。

每一次 操作 ，你可以选择 s 中两个 相邻 的字符，并将它们交换。

请你返回将 s 变成回文串的 最少操作次数 。

注意 ，输入数据会确保 s 一定能变成一个回文串。

```
impl Solution {
    /// 计算将字符串转换为回文串所需的最小相邻交换次数
    ///
    /// 算法思路（贪心 + 双指针）：
    /// - 指针 `i` 从左向右移动，指针 `j` 从右向左移动
    /// - 对于每个位置 `i`，在区间 `(i, j]` 内从右向左查找第一个与 `s[i]` 相同的字符
    ///   - 若找到位置 `k`，则将 `s[k]` 通过相邻交换移动到 `j` 位置，步数为 `j - k`
    ///   - 然后 `j` 左移一位，继续下一轮
    ///   - 若找不到（即 `k == i`），说明 `s[i]` 是出现奇数次的字符，应放到中间位置
    ///     - 此时直接累加 `n/2 - i` 步（移到中间所需交换次数）
    ///     - 注意：此时不移动 `j`，因为中心字符只占一位，其他字符仍需配对
    ///
    /// 时间复杂度 O(n²)，空间复杂度 O(n)
    pub fn min_moves_to_make_palindrome(s: String) -> i32 {
        let mut chars: Vec<char> = s.chars().collect();
        let n = chars.len();
        let mut ans = 0;
        let mut j = n - 1;

        // 贪心处理每个左指针位置
        for i in 0..n / 2 {
            // 从右向左查找第一个与 chars[i] 相同的字符
            let mut k = j;
            while k > i && chars[k] != chars[i] {
                k -= 1;
            }

            if k == i {
                // 未找到配对：当前字符是奇数字符，直接计算移至中间所需的步数
                ans += (n / 2 - i) as i32;
            } else {
                // 找到配对：将 chars[k] 通过相邻交换移动到位置 j
                for t in k..j {
                    chars.swap(t, t + 1);
                    ans += 1;
                }
                j -= 1; // 右指针左移，表示右端已配对
            }
        }
        ans
    }
}
```
